ПРАВИЛО ДЛЯ АГЕНТА: КАРТА АКТИВІВ БІЗНЕС-ГРУПИ

1. Завжди використовуй файл ua_business_map_template.svg як незмінну географічну основу.
2. Не аналізуй і не перемальовуй контури областей. Не змінюй шар map-regions.
3. Для кожної бізнес-групи створи лише JSON за схемою business_group_map_data.example.json.
4. Кожен актив повинен мати:
   - name;
   - type: hq, production, quarry, warehouse, retail, logistics, energy, agriculture або other;
   - region_id з ua_business_map_manifest.json;
   - бажано lat і lon. Якщо координати невідомі, renderer поставить позначку в опорній точці області.
5. За замовчуванням результат має містити тільки карту, підсвічені області, позначки активів та короткі підписи. Не додавай текстові аналітичні блоки, фото, логотипи або окрему інфографіку.
6. Побудова:
   python render_business_map.py data.json output.svg --png output.png
7. Не використовуй для активу координати іншої компанії або іншої бізнес-групи. Перед рендерингом звір назву кожного активу, місто, область та належність до групи.
8. Якщо належність активу або його місце розташування не підтверджені, не наносити його як встановлений факт. Або пропустити, або позначити в JSON як непідтверджений у description.
