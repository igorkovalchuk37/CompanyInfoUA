# Шаблон карти активів бізнес-групи в Україні

Пакет усуває потребу щоразу аналізувати великий SVG. Контури регіонів є незмінною основою, а агент працює лише з компактним JSON.

## Склад пакета

- `ua_business_map_template.svg` — стандартизований SVG із 27 адміністративними контурами України.
- `ua_business_map_manifest.json` — коди областей, українські назви, опорні координати та типи активів.
- `render_business_map.py` — рендерер JSON → SVG; за потреби також PNG.
- `business_group_map_data.example.json` — приклад вхідних даних.
- `business_group_map_data.schema.json` — формальна схема перевірки JSON.
- `AGENT_INSTRUCTION.txt` — готове правило для системних інструкцій агента.

## Що виправлено у вихідному SVG

- `viewbox` замінено на коректний регістр `viewBox`.
- Контури винесені в незмінний шар `map-regions`.
- Для кожного регіону додано стабільний ID, українську й англійську назви.
- Київська область (`UA30`) та місто Київ (`UA32`) розділені однозначно.
- Додано приховані опорні точки регіонів.
- Додано окремі динамічні шари для маркерів, підписів, зв’язків, легенди та заголовка.
- Додано універсальний маркер і фіксовану систему типів активів.
- Додано перетворення широти/довготи у координати SVG.

## Мінімальний JSON

```json
{
  "group_name": "Назва групи",
  "show_title": false,
  "show_legend": false,
  "assets": [
    {
      "name": "Завод",
      "type": "production",
      "region_id": "UA12",
      "lat": 48.0,
      "lon": 33.0,
      "city": "Кривий Ріг"
    }
  ]
}
```

## Запуск

```bash
python render_business_map.py data.json output.svg --png output.png
```

Якщо `lat` і `lon` відсутні, але задано `region_id`, позначка автоматично ставиться в опорній точці області. Кілька активів в одній точці автоматично розсуваються.

## Важливий принцип

Геометрія карти більше не аналізується. Для нової бізнес-групи змінюється лише JSON зі списком підтверджених активів.
